SDK for Javascript¶

Pre-requisites¶

First of all you need to get AEON SDK from GitLab.

Instance the SDK¶

To start publishing/subscribing with the SDK, it si necessary to instanciate a new SDK object.

AeonSDK(url[, subscriptionData])¶
Arguments:
  • url (string) – publish/subscribe url. The constructor will determine if the url is for publish or subcribe and will set its operation mode to “publish” or “subscribe”.
  • subscriptionData (JSON) – Mandatory if you will use the object for subscriptions. It is a JSON containing information about your subscription, this information will be used to retrieve your existing subscription data or to create a new one (if it does not exist or it was deleted).
Returns:

and SDK object

 
{
    "id": "demo-test",
    "desc": "create a new subscription for testing"
}

The combination of “id” and “desc” (strings) makes your subscription unique in the AEON network. Details about persistant subscriptions: Persistent subscription:

Publish¶

This functionality allows to publish a message. The SDK created object has to be in publish mode.

publish(data[, control])
Arguments:
  • data – JSON containing the data to be published.
  • control – callback function to receive AEON control messages.

You can publish without a control callback function. If control callback is passed, controll messages will be received:

  • Code 0: Unknwon error
  • Code 1: Bad URL
  • Code 3: Communication infrastructure down
  • Code 50: Communication infrastructure up
  • Code 101: SDK operating in Subscription Mod

Subscribe¶

This functionality allows to subscribe to an specific channel and receive all the published information. The SDK created object has to be in ssubscription mode.

subscribe(deliveredMessage[, control])
Arguments:
  • deliveredMessage – callback function to receive published messages.
  • control – callback function to receive AEON control messages.

Each time that a message arrives over the channel, the callback deliveredMessage will be executed. You can be subscribed (not recommended) without a control callback function. If control callback is passed, controll messages will be received:

  • Code 0: Unknwon error
  • Code 1: Bad URL
  • Code 3: Communication infrastructure down
  • Code 1: Bad URL
  • Code 50: Communication infrastructure up
  • Code 100: SDK operating in Publication Mode
  • Code 201: Subscription in use
  • Code 203: Subscription incorrect
  • Code 250: You have been subscribed

PauseSubscription¶

This operation will allow to stop receiving data from a subscription.

function pauseSubscription()¶

Specific control messages received through callback control:

  • Code 202: You are not subscribed
  • Code 252: You have been unsubscribed

ContinueSubscription¶

This operation will allow to re-start receiving data from a subscription that was paused.

function continueSubscription()¶

Specific control messages received through callback control:

  • Code 250: You have been subscribed

DeleteSubscription¶

This operation will delete the existing subscription and could not be recovered or continued.

deleteSubscription()¶

Specific control messages received through callback control:

  • Code 202: You are not subscribed
  • Code 251: Your subscription has been deleted
