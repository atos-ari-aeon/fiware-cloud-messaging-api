SDK for Java¶

Instance the SDK¶

To start publishing/subscribing with the SDK, it si necessary to instanciate a new SDK object and retreive a new subscription for your channel.

AeonSDK(String url, String id, String desc)¶
Arguments:
  • url (string) – publish/subscribe url. The constructor will determine if the url is for publish or subcribe and will set its operation mode to “publish” or “subscribe”.
  • id (string) – Mandatory only for subscription mode.
  • desc (string) – Mandatory only for subscription mode.
Returns:

and SDK object

The combination of “id” and “desc” (strings) makes your subscription unique in the AEON network. This information will be used to retrieve your existing subscription data or to create a new one (if it does not exist or it was deleted).

Also, you can use a previously obtained subscription. More details Persistent subscription:

AeonSDK(String url, JSONObject subscription)
Arguments:
  • url (string) – subscribe url.
  • subscription (JSONObject) – previously obtained subscription
Returns:

and SDK object

Publish¶

This functionality allows to publish a message. The SDK created object has to be in publish mode.

publish(JSONObject jsonData[, AEONInterface callback])
Arguments:
  • jsonData (JSONObject) – JSON containing the data to be published.
  • callback (AEONInterface) – callback containing the methods (invoked asynchrounsly) to receive AEON control messages.

See AEON callbacks for details about callbacks.

You can publish without a callback. Control messages received through callback.control:

  • Code 0: Unknwon error
  • Code 1: Bad URL
  • Code 3: Communication infrastructure down
  • Code 50: Communication infrastructure up
  • Code 101: SDK operating in Subscription Mod

Subscribe¶

This functionality allows to subscribe to an specific channel and receive all the published information. The SDK created object has to be in ssubscription mode.

subscribe(AEONInterface callback)
Arguments:
  • callback (AEONInterface) – callback with control and receivedEvent methods.

See AEON callbacks for details about callbacks.

Each time that a message arrives over the channel, the callback.receivedEvent function will be executed. Control messages received through callback.control:

  • Code 0: Unknwon error
  • Code 1: Bad URL
  • Code 3: Communication infrastructure down
  • Code 1: Bad URL
  • Code 50: Communication infrastructure up
  • Code 100: SDK operating in Publication Mode
  • Code 201: Subscription in use
  • Code 203: Subscription incorrect
  • Code 250: You have been subscribed

PauseSubscription¶

This operation will allow to stop receiving data from a subscription.

pauseSubscription()¶

Specific control messages received through callback.control:

  • Code 202: You are not subscribed
  • Code 252: You have been unsubscribed

ContinueSubscription¶

This operation will allow to re-start receiving data from a subscription that was paused.

continueSubscription()¶

Specific control messages received through callback.control:

  • Code 250: You have been subscribed

DeleteSubscription¶

This operation will delete the existing subscription and could not be recovered or continued.

deleteSubscription()¶

Specific control messages received through callback.control:

  • Code 202: You are not subscribed
  • Code 251: Your subscription has been deleted

AEON callbacks¶

In order to manage the events received during publishing/subscribing you have to implement a class based on AEONInterface. This class only contains the methods: receivedEvent (where the published data is received) and control (AEON control messages for errors, networ status, etc..)

public static class MyAEONCallbacks implements AEONInterface {
        @Override
        public void receivedEvent(JSONObject data) {
                System.out.println("Event received" + data.toString());
        }

        @Override
        public void control(JSONObject data) {
                System.out.println("Control Message" + data.toString());

        }

}
