API Users Documentation¶

Documentation for user’s management. Most of the AEON’s functionalities requires registered users. This registered users will make use of the AEON’s functionality to create entities and channels.

You can try all the operation using AEON’s API

Login¶

Logging process with an existing user and password (POST /users/) . If the process results “ok” you will receive a cookie with your session. This cookie will be used in most of the operations.

POST /login/¶

Login process

Example request:

POST /login/ HTTP/1.1
Host: :api-host:`API example host`
Content: application/json
Accept: application/json

{
  'username': 'john',
  'password': 'john',
  'type': 'user'
}

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "code": 200,
  "desc": "ok"
}
Status Codes:
  • 200 – no error
  • 40x – error
  • 400 – Incorrect model received, check mandatory params and syntax
  • 500 – unknown error (contact for support)

Logout¶

Logout process. The user set in the cookie will be unset from it. The cookie won’t be deleted from the server.

GET /logout¶

Logout process

Example request:

GET /logout HTTP/1.1
Host: :api-host:`API example host`
Accept: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "code": 200,
  "desc": "ok"
}
Status Codes:
  • 200 – no error
  • 205 – Unathorized
  • 500 – unknown error (contact for support)

CREATE user¶

Create/Register a new user into the system. By the moment, it only includes information about user and password.

POST /users/¶

Register a new user

Example request:

POST /users/ HTTP/1.1
Host: example.com
Content: application/json
Accept: application/json

{
     "username": "john",
     "password": "john",
     "type": "user",
     "_id": "john"
}

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "code": 200,
  "desc": "ok"
  "result":
 [
  {
     "username": "john",
     "password": "john",
     "type": "user",
     "_id": "john"
  }
 ]
}
Status Codes:
  • 200 – no error
  • 400 – Incorrect model received, check mandatory params and syntax
  • 400 – user already exists
  • 40x – error

User Info¶

Retrieves the information related to the user identified in the cookie.

GET /users/user¶

Get user info

Example request:

GET /users/user HTTP/1.1
Host: :api-host:`API example host`
Accept: application/json

Example response:

  HTTP/1.1 200 OK
  Content-Type: application/json

  {
    "code": 200,
    "desc": "ok",
    "result": [
        {
            "_id": "userID",
            "type": "user",
            "username": "username"
        }
    ]
}
Status Codes:
  • 200 – no error
  • 205 – Unathorized
  • 500 – unknown error (contact for support)

GET User¶

Get complete information of an specific user. Preconditions:
  • In order to get extra information of an existing user you need to be logged
  • You need to be logged as (user_id): (Login)
GET /users/(user_id)¶

Get information for (user_id).

Example request:

GET /users/jhon HTTP/1.1
Host: example.com
Accept: application/json
Query Parameters:
 
  • user_id – user_id email identifier

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "code": 200,
  "desc": "ok",
  "result":
  [
   {
      "username": "jhon",
      "password": "jhon",
      "type": "user"
   }
  ]
}
Status Codes:
  • 200 – no error
  • 401 – You are not authorized for this operation
  • 401 – Not authenticated
  • 40x – error

Remember the user’s password¶

This function is in charge of helping to remember a forgotten or missed password. Mainly, an existing user id will be requested (users are registered with emails). Thus, the user will receive an email with extra information in his email address. The extra information contains a temporal code used to reset the password with (Reset the user’s password).

GET /users/(user_id)/rememberPassword¶

Generate a temporal code to change the user’s password. (user_id) is the email of an existing user. Through this email he will receive the necessary information to (Reset the user’s password).

Example request:

GET /users/user@domain.com/rememberPassword HTTP/1.1
Host: example.com
Accept: application/json
Query Parameters:
 
  • user_id – user_id email identifier

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
 "code": 200,
 "desc": "ok",
 "result": [
   "You will receive an email with information for reset."
 ]
}
Status Codes:
  • 200 – no error
  • 404 – The user does not exist
  • 500 – Error sending reset email

Reset the user’s password¶

This function is in charge of resetting a forgotten or missed password. The user needs to provide an existing user id (users are registered with emails), the new password and a temporal code generated by (Remember the user’s password).

PUT /users/(user_id)/rememberPassword/(code)¶

Changes user’s password of (user_id) using the retrieved (code) by (Remember the user’s password).

Example request:

PUT /users/user@domain.com/rememberPassword/8c5bdbf8-8c2f-46fb-8ae5-b7a579aca07b  HTTP/1.1
Host: example.com
Content: application/json
Accept: application/json

{
     "password": "newPassword",
     "type": "user"
}
Query Parameters:
 
  • user_id – user_id email identifier
  • code – temporal code to reset the password

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "code": 200,
    "desc": "ok",
    "result": [
    ]
}
Status Codes:
  • 200 – no error
  • 404 – user does not exist
  • 500 – error trying to reset user password
